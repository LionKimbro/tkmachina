How Associates Work

An associate is the Little-Castles-side companion to a Tk widget or widget-like resource.

The associate is the boundary object that gives a castle a modeled, data-focused way to work with concrete Tk objects.

Diagram placeholder Castle state -> desired -> associate projection -> Tk widget -> observed/events -> routes.

Desired

associate["desired"] is the projection target. It contains values the castle or runtime wants projected onto the concrete widget.

rt.target_associate(button)
rt.set_desired("text", "Save")

Observed

associate["observed"] is the associate's public modeled-observation surface. It is not a complete mirror of Tk. Each associate type defines which observed fields it intentionally maintains.

entry["observed"]["text"]
entry["observed"]["focused"]

Private

associate["private"] is implementation detail: trace IDs, private Tk variables, projected-value caches, and suppression flags.

entry["private"]["text_var"]
entry["private"]["text_trace"]

Events

Associates emit semantic messages through their outbox. Event interest controls which messages are emitted. It does not control which modeled observed fields are maintained.

Common Widget Events

Reusable binders support opt-in events such as focused, unfocused, pointer_entered, clicked, key_pressed, and configured.
